# 一、产品定位

**类型：** 时间管理 + 游戏化激励 APP

核心机制：

```text
做事情 → 记录时间 → 生成虚拟币 → 可兑换奖励
                     ↓
                   热力图
```

目标：
通过 **可视化进度 + 虚拟奖励系统** 提高用户持续专注。

类似思路的产品：

* Forest
* Habitica

但你的设计更偏：

```text
时间追踪 + GitHub热力图 + 虚拟经济系统
```

---

# 二、核心功能模块

系统主要分为 **5 个模块**：

```text
任务管理
计时系统
热力图统计
虚拟货币系统
兑换商店
```

---

# 三、功能需求

## 1 任务管理（Task）

用户可以创建自己要做的事情。

功能：

* 新建任务
* 编辑任务
* 删除任务
* 任务颜色标记

数据字段：

| 字段          | 说明    |
| ----------- | ----- |
| id          | 任务ID  |
| title       | 任务名称  |
| color       | 热力图颜色 |
| create_time | 创建时间  |
| is_active   | 是否启用  |

示例：

```text
学习
健身
写代码
阅读
```

---

# 四、计时系统

用户开始做某件事时启动计时。

流程：

```text
选择任务
↓
点击开始
↓
计时运行
↓
点击结束
↓
记录时间
```

记录数据：

| 字段          | 说明    |
| ----------- | ----- |
| id          | 记录ID  |
| task_id     | 任务ID  |
| start_time  | 开始时间  |
| end_time    | 结束时间  |
| duration    | 持续时间  |
| coin_earned | 获得虚拟币 |

规则：

```text
1 秒 = 1 coin
```

例如：

```text
30分钟 = 1800 coin
```

---

# 五、热力图统计（核心视觉）

类似 **GitHub贡献图**。

参考：

GitHub

特点：

* 每个格子代表 **一天**
* 颜色表示 **专注强度**
* 显示 **最近 365 天**

颜色等级：

| 等级 | 时间      |
| -- | ------- |
| 0  | 0分钟     |
| 1  | 0-30分钟  |
| 2  | 30-60分钟 |
| 3  | 1-2小时   |
| 4  | 2小时以上   |

数据字段：

| 字段             | 说明    |
| -------------- | ----- |
| date           | 日期    |
| total_duration | 总专注时间 |
| total_coin     | 生成虚拟币 |

---

# 六、虚拟币系统

用户专注时间会生成虚拟币。

规则：

```text
coin = 专注秒数
```

钱包字段：

| 字段           | 说明   |
| ------------ | ---- |
| balance      | 当前余额 |
| total_earned | 累计收入 |
| total_spent  | 累计消费 |

功能：

* 自动增加虚拟币
* 兑换商品
* 查看历史

---

# 七、兑换商店

用户可以 **自定义奖励**。

例如：

```text
喝奶茶
打一小时游戏
看电影
买耳机
```

商品字段：

| 字段          | 说明   |
| ----------- | ---- |
| id          | 商品ID |
| name        | 商品名称 |
| price       | 价格   |
| description | 描述   |
| create_time | 创建时间 |
| is_active   | 是否上架 |

---

# 八、兑换记录

用户兑换奖励后生成记录。

字段：

| 字段          | 说明   |
| ----------- | ---- |
| id          | 记录ID |
| item_id     | 商品   |
| price       | 花费   |
| create_time | 时间   |
